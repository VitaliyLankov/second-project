Поведением блока при переполнении управляет свойство overflow. Это свойство контейнера, а не текстового элемента. Мы ещё вернёмся к нему, когда будем говорить о переполнении контейнеров. Но сейчас без него не получится выполнить задумку. Свойство overflow назначается родительскому элементу, в который не помещается содержимое. Это содержимое можно скрыть значением hidden или убрать за скроллбар значением scroll. Подробнее про все режимы прочитайте в Доке, а два упомянутых попробуйте на примере.

Для вёрстки блока, не привязанного к окружающему его контенту, используйте тег article. Такой блок можно убрать со страницы и вставить в любое другое место сайта. В спецификации у этого тега довольно расплывчатое описание. Из-за этого не всегда понятно, когда использовать, а когда section. Чтобы ответить на этот вопрос, мысленно проведите эксперимент. Уберите этот блок со страницы. Если ничего не изменилось, смысл не потерялся, то можно использовать article.

Если нужно процитировать целую фразу, то используйте тег blockquote. У этого тега есть не обязательный, но полезный атрибут cite. Он позволяет указать автора цитаты.

Все о флексе

У нас есть блок с тремя вкладками. Кнопки для переключения между вкладками выстроились в одну строку. Нужно сделать так, чтобы кнопки для переключения вкладок перестроились в столбец — одна под другой. Чтобы добиться такого эффекта, достаточно изменить направление основной оси — она должна смотреть не слева направо, а сверху вниз. И поможет это сделать свойство flex-direction. Добавьте флекс-контейнеру .tabs свойство flex-direction: column и посмотрите, как изменится отображение вкладок.

raw-gap управляет расстоянием между рядами

column-gap управляет расстоянием между столбцами

Когда у нас стоит wrap на куче элементов флекса, которые не помещаются в контейнер без врапа, управлять их расположением по горизонатли align-items не получится, тк это типа как одна строка все. Для этого есть align-content. Пример выше

Порядок элементов в флексе. Order 1, order 2, order 3. Задаестя в css

align-self - когда нужно одному элементу придать особый порядок. Flex-grow:1 когда нужно, чтобы элемент занял все свободно пространство

Решить эту задачу поможет свойство flex-basis. Оно указывает размер элемента до того, как на него начнёт действовать расширение. Значением flex-basis может быть размер в любых относительных или абсолютных единицах. Значение по умолчанию — auto. В таком случае элемент занимает столько пространства, сколько требуется для отображения его контента.
